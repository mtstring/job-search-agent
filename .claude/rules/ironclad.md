# 鉄則

これらは「すべての操作に優先」する不変ルール。迷ったらこのファイルに戻る。

## スケジュール

- **スケジュールの正は Google Calendar のみ**
- セッションログ・メモリ・memo に日程を書かない
- 日程確認は **必ず** Google Calendar API を叩く（推測・記憶に頼らない）

## メール（Gmail）

- **下書きは本文のみ**。件名・ラベル・注記・ヘッダー・システムテキストを含めない（複数回指摘済み）
- **返信は全員返信（Reply-All）**。To / CC の宛先を漏れなく含める
- **メール検索で `is:unread` を使わない**。既読でも未対応のメールがある
- **返信には元メール本文の引用を下につける**。引用は純粋な元メール本文のみ。AI 指示文・内部メモ・システムテキストを絶対に含めない

返信本文の構成:

```
{自分の返信本文}

{送信者名} <{メールアドレス}> {日時}:
> {元メールの本文（純粋なメール本文のみ）}
```

## 発言

- **面談・面接で「会社のキャッシュ不足・倒産リスク」を転職理由として絶対に言わない**

## ファイル命名

- **書き起こしファイル**: 先頭に会社名をつける
- **`unknown/` フォルダは使わない**
- **会社ディレクトリの slug は英語のみ**（例: `example-corp`, `acme-tech`）
  - 日本語 slug 禁止（✗ `エグザンプル株式会社` → ○ `example-corp`）
- **書き起こし DL 時に日付プレフィックスを追加しない**（ユーザーが手動付与済み）

## 曜日・日付

- 曜日を推測で書かない。カレンダー API か `date` コマンドで確認する

## 選考発言の戦略

- 年収・意思決定者で絞り込み、断りも入れる（2026-04-14 決定）
